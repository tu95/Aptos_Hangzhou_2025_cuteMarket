✅ cuteMarket 前端设计说明（Cursor 版）
目标：用 React + Aptos SDK 写一个 Web 页面，让用户连接钱包、查看内置投注项目、下注。

1. 页面结构（3个核心页面）
首页：列出所有内置投注项目（卡片形式）
项目详情页：展示选项、投注池、下注表单
连接钱包按钮：全局固定在右上角
2. 内置项目（固定，硬编码）
项目共 5 个，直接写死在前端：

3. 功能需求
3.1 连接钱包
使用 @aptos-labs/wallet-adapter-react
页面加载时自动检测钱包连接状态
显示“连接钱包”按钮 → 点击弹出钱包选择（Martian/Pontem）
成功连接后显示用户地址（截断：0x1234…abcd）
3.2 展示项目列表（首页）
遍历内置项目数组
每个卡片显示：
项目名称
选项（如：下台 / 继续呆着）
结束时间（格式：YYYY-MM-DD）
状态：
未到时间 → “开放中”
已过时间 → “已关闭”（但未结算）
已结算 → “已开奖” + 获胜选项（先写“待结算”）
📌 当前不从链上读取投注池，全部显示为 “0 APT”（后续再接入合约） 

3.3 项目详情页
点击卡片进入 /project/:id
显示：
项目名称、描述（可选）
每个选项的按钮（蓝色，可点击）
每个按钮旁显示当前投注额（先写 “0 APT”）
输入框：输入下注金额（最小 1 APT）
按钮：“下注 1 APT”
点击“下注”：
检查金额 ≥1 APT
检查钱包已连接
调用 signAndSubmitTransaction，调用合约 place_bet(projectId, optionIndex, amount)
3.4 状态提示
如果项目已过期（当前时间 > 结束时间）：
显示：“已截止，等待开奖”
禁用下注按钮
如果已结算：
显示：“开奖结果：[获胜选项]”
显示：“你赢了！” 或 “很遗憾”
4. 技术栈
框架：React + Vite
钱包：@aptos-labs/wallet-adapter-react + @aptos-labs/wallet-adapter-ant-design
链交互：aptos SDK（v1.12+）
样式：Tailwind CSS（简单、快速）
状态管理：React Context 或 useState（无需 Redux）
